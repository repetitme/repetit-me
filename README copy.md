# Проект: "RepetitMe"

**Стек технологий**: React, Redux, TypeScript, SCSS, Vite, HTML, ESLint, Prettier, Stylelint.

**Описание работы приложения**: В приложении RepetitMe ученики смогут найти себе репетитора по конкретному учебному предмету, записаться на занятие, написать поделиться впечатлениями о прошедшем уроке и о самом репетиторе. Репетиторы, в свою очередь, создают подробный профиль, с заполнением информации о себе и о своем образовании, подтверждая его документом. Они могут принимать заявки от учеников и запланировать урок на любое время.

## Структура проекта:

- `src`: Корневая директория проекта, в которой содержится основная логика приложения.
- `src/app`: Директория, в которой расположены главный файлы приложения для входа в него, а так же его состояние.
- `src/assets`: Директория, в которой расположены статические файлы, такие как картинки, стили, шрифты и другие ресурсы.
- `src/config`: Директория, в которой расположены конфигурационные файлы для приложения.
- `src/entities`: Директория, в которой расположены сущности приложения, такие как пользователи, уроки и другие объекты.
- `src/features`: Директория, в которой расположены переиспользуемые функции для работы приложения.
- `src/pages`: Директория, в которой расположены компоненты страниц приложения.
- `src/shared`: Директория, в которой расположены переиспользуемые компоненты, хуки, утилиты и типы для приложения.
- `src/widgets`: Директория, в которой расположены самостоятельные компоненты, которые можно использовать в любой точке приложения.

## Установка и запуск приложения:

Для установки и запуска проекта необходимо выполнить команды для установки зависимости и запуска проекта в режиме разработки, соответственно.

```
npm install
npm run dev
```

или

```
yarn
yarn dev
```

## Сборка

Для сборки проекта в продакшн необходимо выполнить команду сборки в режиме разработки.\
Для данного проекта нет ограничений по использованию пакетного менеджера, поэтому можно использовать также YARN или PNPM.

```
npm run build
```

или

```
yarn build
```

## Архитектура проекта:

Приложение построено на архитектуре **FSD (Feature-Sliced Design)**, что упрощает организацию кода и его поддержку. Этот подход позволяет разделить приложение на независимые функциональные модули, что способствует лучшей управляемости и масштабируемости. [Подробнее об архитектуре](https://feature-sliced.design/docs).
